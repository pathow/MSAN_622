Graph of Letter Counts

Homework 1 App

Enter your string of interest in the box below and see the counts of each letter that appears!

Features Added

C-level Functionality

  • Gets the query string from the URL
  • Stores the data as an array of objects, sorted and ordered by the values of letter count
  • Altered the axis line to make it less thick
  • Made tick marks conditional -- a tick for each unit up to the maximum value, unless the maximum value is > 10, then only 10 ticks appear
  • Set the colors of the bars, in this case to a gradient where the higher the relative value, the darker the color

B-level Functionality

  • Added text entry box that refreshes the page with a inputted string -- works both on click and pressing enter
  • Bars change color on mouse hover (Note: added effect beyond the tutorial that didn't count...now transition effect too so that color lingers after mouse moves away to more gently fade out)
  • Text stating counts of the bars appears on mouseover

A-level Functionality

  • Upon new graph creation, bars are animated to appear to grow from 0, in order of values

Homework Writeup

As an overall project, I have implemented many features on to this chart that should demonstrate a strong grasp of D3. One of the main things that I was deliberate about was to keep the visualization clean and not aesthetically confusing. It is for this reason that I decided to implement a hard-coded sort of the data into my Javascript code instead of pursuing one of the options of making a changing sort function (i.e., implementing a button or checkbox for different sorts. I also believe that coding in only a single sort option myself gives me an extra degree of control over being absolutely certain that the visitor to the page is seeing exactly what I want them to see. In particular, by ording from maximum to minimum letter counts, and shading each bar darker for more counts, I am explicitly drawing the attention towards what letters appear most in the string.

This is not to say though that I shun interactivity, though. My main interactive focus was first on allowing the user to change the string being parsed and counted on their own through a more intuitive mechanism of a text entry box instead of simply relying on the query string method. The other dimension I added to interactivity involves actions that occur on mouseover in the bars of the graph -- unobtrusive mouseover text appears showing the counts of that letter, and also the bars change color in a controlled animation that makes for more fluid transitions as a user moves from bar to bar.
